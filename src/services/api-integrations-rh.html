Pourquoi intégrer CVérifié directement dans votre ATS ?

Lancer une vérification de CV manuellement depuis le portail CVérifié prend 5 minutes par dossier. Pour une équipe RH qui traite 50 recrutements par mois, cela représente plus de 4 heures de saisie manuelle. L'API CVérifié automatise entièrement ce processus et intègre les résultats directement dans votre workflow.

Déclenchement automatique à chaque candidature retenue

Configurez un déclencheur dans votre ATS : dès qu'un candidat passe en phase d'entretien (ou à l'étape de votre choix), la vérification CVérifié se lance automatiquement. Plus besoin d'action manuelle du recruteur. Le gain de temps est immédiat et le processus devient systématique.

Résultats directement dans votre interface

Via webhook, CVérifié envoie le rapport JSON dès que la vérification est terminée. Votre intégration peut afficher le statut du rapport directement dans la fiche candidat de votre ATS : diplôme confirmé, expériences vérifiées, score de références — sans quitter votre interface.

Consentement RGPD géré par l'API

L'API CVérifié gère intégralement le flux de consentement : l'email d'information au candidat, la page de consentement, la confirmation horodatée. Vous n'avez qu'à passer l'adresse email du candidat dans votre requête API — tout le reste est pris en charge conformément au RGPD.

ROI immédiat : 2 h de travail manuel économisées par dossier

En tenant compte de la saisie manuelle, du suivi des délais, de la relance des candidats et du téléchargement des rapports, l'automatisation via API économise environ 2 heures de travail administratif par dossier. Pour 50 dossiers par mois, c'est 100 heures récupérées par l'équipe RH.

Fonctionnement technique

Comment fonctionne l'API CVérifié ?

Trois appels API suffisent pour déclencher une vérification et en récupérer les résultats. Le flux est asynchrone : vous soumettez la demande, CVérifié traite et vous notifie via webhook à la livraison du rapport.

1

POST /verifications — Soumission du dossier

Envoyez une requête POST avec les données du candidat (nom, email, diplômes et/ou expériences à vérifier). CVérifié déclenche le processus de consentement et retourne un verification_id. Délai de réponse API : moins de 200 ms.

2

CVérifié traite la vérification (48 h)

Après accord du candidat, CVérifié mène les vérifications auprès des sources officielles. Pendant ce temps, vous pouvez interroger GET /verifications/{id} pour suivre l'état en temps réel (pending, consent_required, in_progress, completed).

3

Webhook POST — Résultats JSON livrés

À la clôture du rapport, CVérifié envoie un POST vers votre URL de webhook avec le rapport complet en JSON (ou un lien vers le PDF). Le payload inclut le statut de chaque vérification et le lien de téléchargement du rapport PDF signé.

Spécifications techniques

Quelles sont les fonctionnalités techniques de l'API CVérifié ?

L'API CVérifié est conçue pour être simple à intégrer et robuste en production. Elle suit les standards REST modernes et est documentée avec OpenAPI/Swagger.

Authentification OAuth2 / API Key

Deux modes d'authentification disponibles : OAuth2 (recommandé pour les intégrations complexes avec scopes) et API Key simple (pour les intégrations légères). Les clés API sont générées depuis votre tableau de bord CVérifié et révocables à tout moment.

Webhooks de notification

CVérifié envoie un POST vers votre URL de webhook à chaque changement d'état majeur : consentement reçu, vérification en cours, rapport livré. La signature HMAC-SHA256 de chaque webhook permet de vérifier l'authenticité de la notification.

Sandbox de test disponible

Un environnement sandbox complet est disponible pour tester votre intégration sans créer de vrais dossiers ni engager de frais. Le sandbox simule tous les cas de figure : consentement accordé, refus, délais variables, discordances détectées. Accès immédiat depuis votre compte.

Documentation OpenAPI / Swagger

Documentation complète et interactive disponible en ligne. Chaque endpoint est documenté avec exemples de requêtes et de réponses, codes d'erreur, et tests exécutables directement depuis la documentation Swagger UI.

SLA : 99,9 % d'uptime garanti

L'API CVérifié dispose d'un SLA de 99,9 % d'uptime mensuel. En cas d'incident, la page de statut (status.cverifie.fr) est mise à jour en temps réel. Le temps de réponse de l'API (hors traitement de la vérification) est inférieur à 200 ms pour 95 % des requêtes.

Gestion du consentement RGPD intégrée

Le flux de consentement est entièrement géré par l'API. Vous passez l'email du candidat ; CVérifié envoie l'email d'information, héberge la page de consentement conforme RGPD, et retourne dans le rapport la preuve de consentement horodatée et archivée.

Questions fréquentes

Questions fréquentes sur l'API CVérifié

Faut-il être développeur pour utiliser l'API CVérifié ?

L'API CVérifié est destinée aux équipes techniques (développeurs, DSI) qui souhaitent intégrer la vérification de CV dans leur ATS ou SIRH. Elle requiert des compétences en développement backend pour effectuer des requêtes HTTP et gérer des webhooks. Cependant, si vous utilisez Greenhouse ou un ATS avec un marketplace d'intégrations, l'installation peut se faire en self-service sans développement. Pour toute autre intégration, un développeur ayant quelques heures disponibles suffira pour connecter l'API CVérifié à votre système.

L'API CVérifié est-elle disponible en sandbox pour les tests ?

Oui. Un environnement sandbox complet est disponible gratuitement depuis votre espace client CVérifié. Le sandbox utilise des données fictives et simule des délais réduits (quelques minutes au lieu de 48 h) pour faciliter les tests. Il reproduit tous les scénarios possibles : consentement accordé ou refusé, rapport avec confirmation, rapport avec discordances, timeout de candidat, etc. Aucun frais n'est engagé dans le sandbox. L'accès au sandbox est disponible immédiatement après création de compte.

Comment l'API gère-t-elle le consentement RGPD du candidat ?

Quand vous soumettez une vérification via l'API avec l'adresse email du candidat, CVérifié envoie automatiquement un email d'information au candidat, lui expliquant qu'une vérification est en cours, pour quel employeur, et sur quels éléments de son parcours. Un lien lui permet de donner ou refuser son consentement via une page sécurisée hébergée par CVérifié. La preuve de consentement (horodatée, avec adresse IP) est incluse dans le rapport final. Si le candidat refuse ou ne répond pas dans les 48 h, la vérification est annulée et vous êtes notifié via webhook. Aucun frais n'est prélevé pour les dossiers annulés avant consentement.

L'API peut-elle s'intégrer à n'importe quel ATS ?

L'API CVérifié est une API REST standard qui peut s'intégrer à tout ATS supportant les webhooks sortants ou les intégrations via HTTP. La grande majorité des ATS modernes (Greenhouse, Lever, Workday, BambooHR, Personio, Recruitee, SmartRecruiters, etc.) supportent ce type d'intégration. Si votre ATS est plus ancien et ne supporte pas les webhooks, il est souvent possible de passer par un outil d'automatisation intermédiaire (Zapier, Make/Integromat, n8n) pour faire le pont entre votre ATS et l'API CVérifié sans développement spécifique.

Quel est le délai de réponse de l'API CVérifié ?

L'API CVérifié répond à toutes les requêtes de soumission et de statut en moins de 200 ms (p95). Ce délai correspond au temps de réponse de l'API elle-même, pas au délai de traitement de la vérification. La vérification effective (contact des sources, rapport) prend 48 à 72 heures selon le type de vérification commandée. Vous êtes notifié de la livraison du rapport via webhook — vous n'avez donc pas à poller l'API en boucle pour suivre l'avancement.

Y a-t-il des coûts supplémentaires pour l'accès à l'API ?

Non. L'accès à l'API CVérifié est entièrement gratuit. Il n'y a pas de frais d'inscription, pas de frais mensuels d'accès à l'API, pas de frais de mise en place, et pas de coût minimum. Vous ne payez que les vérifications réellement effectuées, au même tarif unitaire que le service web (14,90 € à l'unité, avec dégressif à partir de 10 dossiers par mois). Le sandbox est également gratuit et illimité. Pour les grandes entreprises avec des volumes importants (plus de 50 dossiers par mois), des tarifs négociés sont disponibles sur demande.
